# ZZZ 角色文件（char:）全局标准化 Changelog

> **日期**: 2026-04-25  
> **范围**: `/世界书/ZZZ/char:*.yaml` — 共 85 个角色文件  
> **目标**: 将所有角色资料统一为轻量化、无剧透、Token 友好的标准 YAML 格式，为沙盒 RP 建立干净的基线环境。

---

## Phase 1：文件格式统一

### 1.1 扩展名重命名
- **操作**: 将所有 `.txt` 角色文件统一重命名为 `.yaml`
- **影响**: 153 个文件
- **脚本**: `format_standardizer.mjs`

### 1.2 残缺 JSON → 标准 YAML 转换
- **操作**: 修复结构性 JSON 语法错误（多根节点、未转义换行、缺失逗号等），转换为纯文本 YAML 格式
- **影响**: 9 个文件
- **脚本**: `fix_json_to_yaml.mjs`
- **特殊处理**:
  - `char:伊瑟爾德.yaml` — 纯中文键名的 JSON，脚本无法自动处理，**手动完整重建**
  - `char:绒枢.yaml` — 内部双引号冲突（`"思考"完美世界"`），改用中文括号引号 `『』` 解决

### 1.3 空文件恢复
- **操作**: 从 git 历史或 `nova-creator-cli` 源目录恢复被误清空的文件，直接转为标准 YAML
- **影响**: 2 个文件
  - `char:狛野真斗.yaml` — 从 git HEAD 恢复
  - `char:蕾.yaml` — 从 git HEAD 恢复

## Phase 2：内容清洗

### 2.1 全领域剧透清除
- **操作**: 扫描所有叙事字段（背景故事、战斗专长、人际关系等），删除涉及法厄同主线未来剧情的描述
- **影响**: 62 个文件
- **脚本**: `apply_lore_patch_v2.mjs`
- **清除目标**: 包含"法厄同"等关键词的剧透性描述行

### 2.2 Token 极致优化
- **操作**: 删除所有 `暂无记录`、`待在沙盒RP中经历与探索` 等占位废话；清空无内容的空壳区块标题
- **影响**: 63 个文件
- **脚本**: `clean_tokens.mjs`

## Phase 3：格式规范化

### 3.1 英文标题 → 中文标题
- **操作**: 将英文区块标题统一为中文标准标题
- **影响**: 9 个文件（仪绛、千夏、南宫羽、塞西莉亚、照、爱芮、琉音、葉瞬光、葉釋淵）
- **脚本**: `fix_format.mjs`
- **映射**:

| 原标题 | 新标题 |
|--------|--------|
| `【name】` | `【基本信息】` |
| `【appearance】` | `【外貌特征】` |
| `【personality】` | `【性格特点】` |
| `【background】` | `【背景故事】` |
| `【special_abilities】` | `【战斗专长】` |
| `【relationships】` | `【人际关系】` |
| `【dialogue_examples】` | `【自己的台词语气】` |

### 3.2 英文字段名 → 中文字段名
- **操作**: 同上 9 个文件中的英文字段名替换
- **映射**: `gender` → `性别`、`hair` → `发型`、`eyes` → `瞳色`、`outfit` → `服饰风格`、`vibe` → `气质描述`、`traits` → `性格类型`、`likes` → `喜好`、`dislikes` → `厌恶`、`identity` → `身份背景`、`achievements` → `重要事件` 等

### 3.3 人际关系格式修复
- **操作**: 将 `name:nature:xxx, description:yyy` 格式统一为 `name:xxx - yyy`
- **影响**: 9 个文件
- **脚本**: `fix_format.mjs`

### 3.4 散文式文件 → 标准 YAML
- **操作**: 将完全没有标题结构的 Wiki/散文格式文件手动重写为标准 YAML
- **影响**: 4 个文件
  - `char:fairy.yaml` — 从纯散文描述重构为【性格特点】结构
  - `char:格莉丝.yaml` — 从 Tab 分隔的 Wiki 格式重构
  - `char:浅羽悠真.yaml` — 从 Tab 分隔的 Wiki 格式重构
  - `char:拉米尔.yaml` — 从残缺 JSON 格式完整重构

### 3.5 `name:` 裸开头 → `【基本信息】` 标准头
- **操作**: 将以 `name: xxx` 开头的文件改为 `【基本信息】` + `姓名:xxx`，孤儿别号行收编为 `别号:` 字段，并将错放在【外貌特征】下的基本信息字段（性别、种族、年龄等）移到【基本信息】中
- **影响**: 7 个文件（仪绛、塞西莉亚、照、爱芮、琉音、葉瞬光、葉釋淵）
- **脚本**: `fix_headers.mjs`

## Phase 4：数据恢复

### 4.1 `[object Object]` 战斗专长恢复
- **问题**: 之前的 JSON→YAML 转换脚本对 `special_abilities` 数组调用了 `.toString()`，导致所有嵌套对象变成 `[object Object]` 垃圾数据
- **操作**: 从 `nova-creator-cli/世界书/ZZZ/` 原始源文件中提取正确的战斗专长数据并回填
- **影响**: 8 个文件
  - 自动恢复（5）: 仪绛、南宫羽、照、琉音、葉釋淵
  - 手动恢复（3）: 塞西莉亚、爱芮、葉瞬光（因原始 JSON 第2行语法错误导致 `eval` 解析失败）
- **脚本**: `fix_abilities.mjs`（自动部分）

### 4.2 杂项手动修复
- `char:蒂娜.yaml` — 修复行首垃圾字符 `e    "肤色"` → `肤色:`，清除行尾不可见字符
- `char:南宫羽.yaml` — 用户手动全面修复（英文标题→中文、`[object Object]`→完整战斗专长、关系格式修正）
